MAC 系统的启动过程和系统启动时运行shell 脚本

Posted on 2014-06-09 | In Mac OS X |

一、 Mac 开机启动脚本（系统启动时运行shell）

1.在开机启动的时候可以通过图形化界面来设置：
system prefernce–>Accounts–>LogItems–>add

2.把shell放到/Labrary/Startupitems/目录中。

3.创建/etc/rc.local文件,把脚本写在文件中。

4.只启动shell,还有一种:
1) 把Terminate 设置为开机启动。这样当系统启动时，可以自动运行shell。
2) 在用户目录下创建或修改”.profile”。当shell 启动时，先运行”.profile” 文件
3) 在”.profile” 中调用想要执行的shell 脚本

5) 应用Mac的launchd,可以参见:地址

注： 苹果的安全检查：用上面说过的2，3 方法时，必须设置文件的用户id 和组id，还有文件的运行权限。具体方法，下文有述。
当设置完成后，可以在终端上用SystemStarter 来调试配置的信息
命令为：SystemStarter -n -D
根据提示的信息可以查看配置的问题。

二、 解决“Insecure Startup Item disabled.错误”

可以把想要启动的shell 放到/Labrary/Startupitems/ 目录中。

必须改变文件的用户id 和组id 才可以，并且权限为755

sudo chown -R root:wheel /Library/StartupItems/xxx

sudo chmod -R 755 /Library/StartupItems/xxx

(如果运行的是shell 文件，则命令中不用加 -R 选项)
